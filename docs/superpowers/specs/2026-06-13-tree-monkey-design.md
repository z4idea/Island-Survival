# 树上猴子设计

## 目标

新增一种不会主动攻击玩家的猴子事件。部分普通树上藏有猴子，玩家可以从露出的尾巴提前识别。玩家身体触碰树后，猴子随机偷走一种当前持有的资源或货币并快速逃跑；玩家在猴子逃脱前击杀它，可取回本次被偷物品。

## 范围

- 仅普通 `tree` 节点可能藏猴，`palm` 和其他节点不参与。
- 约 8% 的普通树藏猴。
- 每棵藏猴树最多触发一次。
- 猴子不攻击玩家、不进入普通动物仇恨与重生流程。
- 本功能不新增存档字段，不改变资源节点 ID、世界生成循环顺序或现有存档版本。

## 生成与确定性

藏猴判定由世界种子和树节点 ID 派生，不额外消耗 `worldgen` 的随机数序列。这样不会改变现有节点与动物的生成结果，也能保证同一种子下藏猴树位置稳定。

树运行时状态记录：

- 是否藏猴。
- 是否已经触发。
- 尾巴对应的 Pixi Graphics。

死亡重生、离开区域或猴子逃脱后，已触发的树不会再次生成猴子。该状态仅在当前游戏运行期间保留；重新载入存档后按种子重新建立，符合现有运行时事件不入存档的模式。

## 视觉与触发

藏猴树在树冠边缘绘制一条棕色弯曲尾巴，并加入轻微摆动动画。猴子本体在触发前不可见。

触发条件是玩家与树中心的距离进入树碰撞半径附近，而不是攻击或砍伐树。触发时：

1. 将树标记为已触发并隐藏尾巴。
2. 从玩家当前数量大于 0 的候选物品中等概率选一种。
3. 扣除该物品当前数量的 10%，向上取整且最少为 1。
4. 在树旁生成猴子，显示偷取提示。
5. 猴子立即沿远离玩家的方向逃跑，并加入少量随机偏角，避免所有逃跑轨迹完全一致。

候选物品包括：

- 基础资源：木材、石头、浆果、肉、兽皮。
- 货币：银币、金币、钻石。

若玩家所有候选物品均为 0，猴子仍会跳下并逃跑，但不携带赃物，提示“什么也没偷到”。

## 猴子实体

新增独立的猴子实体模块，不复用通用 `Animal` 状态机。猴子只负责：

- Pixi 程序化绘制和位置更新。
- 沿固定逃跑方向高速移动。
- 接受现有近战、远程、冥火、闪电和圣翼冲刺伤害。
- 记录被偷物品的类型与数量。
- 击杀返还或跑远消失。

猴子没有主动攻击、追击、普通掉落、金币掉率、成长系数或重生记录。它使用动物碰撞组，使玩家和不可穿越墙体能正常阻挡；逃跑方向在生成时确定，必要时可沿障碍轻微滑动，避免原地卡住。

猴子从触发树移动约 18 个世界单位后视为逃脱，移除物理体和显示对象，赃物永久损失。

## 战斗与返还

猴子应能被所有现有玩家伤害来源命中。为避免在每个伤害入口长期复制特判，提供统一的可伤害目标接口或最小公共方法，使普通动物和猴子都能暴露位置、半径、存活状态及 `damage` 行为。

猴子死亡时：

- 不执行普通动物死亡掉落。
- 将记录的赃物直接返还玩家。
- 更新对应资源或货币 HUD。
- 显示“夺回”浮字和反馈特效。
- 从猴子实体列表、物理世界和 Pixi 容器中移除。

如果猴子没有偷到物品，死亡只播放普通命中与消失反馈。

## 集成位置

- `src/game.ts`：给世界树节点增加藏猴运行时状态；检测玩家触树；维护猴子列表并在主循环物理步进前更新；在重生或销毁流程中清理猴子。
- `src/entities/monkey.ts`：猴子视觉、逃跑、受伤、返还和销毁。
- `src/entities/player.ts`：增加统一的物品扣除与返还入口，确保资源和货币 HUD 同步。
- 现有伤害入口：扩展目标遍历以包含猴子，并保持原有 `meleeImmune`、`latched` 等普通动物过滤规则不变。

不把猴子加入 `ANIMALS`、`WorldData.spawns` 或 `spawnRecords`，避免进入普通动物 AI、掉落和重生机制。

## 测试与验证

先为可独立测试的纯逻辑编写失败测试，再实现：

- 仅从当前数量大于 0 的资源和货币中选择。
- 偷取数量为当前数量的 10% 向上取整，最低 1。
- 全部为 0 时返回空赃物。
- 击杀返还正确类型和数量。
- 达到 18 格逃跑距离后判定逃脱。
- 同一世界种子和树 ID 的藏猴判定稳定，命中率阈值为 8%。

项目当前没有测试脚本，因此实现计划会优先提取不依赖 Pixi/Rapier 的纯函数，并使用与 Node 16 兼容的最小测试方式。完成代码修改后按项目约定运行 `npm run check`，不执行完整构建。
